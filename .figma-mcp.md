## 2. Design Tokens — Nguồn gốc và vị trí

### 2.1 File chứa tokens
```
src/app/globals.css     ← SINGLE SOURCE OF TRUTH cho tất cả tokens
```

**Format:** Tailwind CSS v4 `@theme { }` block — CSS custom properties.

```css
/* globals.css */
@theme {
  --color-primary: #2D90AF;       /* brand/600 */
  --font-sans: var(--font-be-vietnam-pro);
  --font-display: var(--font-playfair-display);
  --spacing-md: 16px;
  --radius: 0.375rem;
}
```

**KHÔNG CÓ** `tailwind.config.js` — mọi override đều qua `globals.css`.  
**KHÔNG SỬA** dòng `@source ".."` ở đầu file.

### 2.2 Token Naming Convention

| Loại | Pattern | Ví dụ |
|------|---------|-------|
| Color primitive | `--color-{scale}-{shade}` | `--color-brand-600` |
| Color semantic | `--color-{role}` | `--color-primary`, `--color-foreground` |
| Font family | `--font-{name}` | `--font-sans`, `--font-display` |
| Font size | `--font-size-{name}` | `--font-size-display-xl` |
| Line height | `--leading-{px}` | `--leading-56` |
| Spacing | `--spacing-{name}` | `--spacing-md` |
| Radius | `--radius-{name}` | `--radius-btn` |
| Animation | `--animate-{name}` | `--animate-fade-in` |

### 5.2 Component Architecture

**Pattern:** CVA (class-variance-authority) cho variants.

```tsx
// src/components/ui/button.tsx
const buttonVariants = cva(
  "inline-flex items-center justify-center ...",
  {
    variants: {
      variant: {
        default: "bg-primary text-primary-foreground hover:bg-primary/90",
        cta: "text-white font-semibold rounded-[var(--radius-btn)] ...",
      },
      size: { default: "h-9 px-4 py-2", lg: "h-10 px-6", sm: "h-8 px-3" }
    }
  }
)
```

**Khi implement từ Figma:**
- Map Figma component → existing shadcn/ui component trước
- Chỉ tạo component mới nếu không có match
- Dùng `cn()` từ `src/lib/utils.ts` để merge classes
